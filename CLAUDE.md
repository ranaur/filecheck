### Core CLI Usage (via `filecheck.py`)
The primary operations are managed through distinct subcommands:

1.  **Generate Manifest**: Creates a new manifest by traversing the target directory, recording metadata and hashes.
    ```bash
    python filecheck.py generate <DIRECTORY> [OPTIONS]
    ```
2.  **Update Manifest**: Re-scans an existing directory. It is optimized to only recompute hashes for files whose metadata (size/mtime/ctime/atime) has changed, improving performance over a full regeneration.
    ```bash
    python filecheck.py update <DIRECTORY> [OPTIONS]
    ```
3.  **Check Manifest**: Compares the current live state of the directory against the saved manifest to report any discrepancies (new items, deleted files, or hash/metadata mismatches).
    ```bash
    python filecheck.py check <DIRECTORY> [OPTIONS]
    ```

*   **Python Compatibility**: The codebase contains artifacts related to **Python 2**. Ignore any analysys in `filecheck-python2.py` module. It is there only for historical purposes.
